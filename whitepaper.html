Abstract

QSCoin is a next-generation cryptocurrency designed to remain secure in a post-quantum world, combining hybrid post-quantum signatures with zk-proofs for privacy and scalability, VRF-based randomness for fairness, and a community-first token economy.

Max Supply: 81,000,000 QSC · Ticker: QSC · Consensus (R&D): Hybrid PoS + PQC · Privacy: Opt-in zk-SNARK notes

1. Vision & Principles

Build a chain that is future-proof against quantum adversaries while offering modern UX and privacy controls. QSCoin prioritizes security, verifiability, credible neutrality, and responsible rollouts.

  • Quantum-safe by default, compatibility via hybrid signatures.
  • Community-first: transparent supply, public roadmap, open development.
  • Optional privacy with clear compliance paths.
  • Lean base layer + rollup-friendly design for scale.

2. Cryptography & Security

2.1 Hybrid Signatures

  • PQC: CRYSTALS-Dilithium (levels tuned per mainnet audits).
  • Classical: ECDSA/secp256k1 for ecosystem compatibility.
  • Address format bundles both public keys; transactions carry both signatures.

2.2 Randomness (VRF)

  • Validator selection with verifiable random functions.
  • Bias-resistant, publicly verifiable randomness beacons.

2.3 Audits & Bug bounty

Third-party audits precede major releases; public bug bounty with severity tiers.

3. Consensus & Protocol

Hybrid Proof-of-Stake with PQC-backed keys, fast finality, and rollup-friendly fork choice.

Block time~3–6s
Finality< 2 min
Tx size (hybrid)+~1.5–2 KB
VRF epoch~1,000 blocks

Node Roles

  • Validators: stake QSC, produce/attest blocks.
  • Full nodes: verify all rules & signatures.
  • Light clients: sync via succinct proofs.

5. Privacy (zk)

Privacidad opt-in vía zk-SNARKs (Groth16/PLONK). Modo público y “notas” privadas. Claves de visualización (view keys) y divulgación selectiva para cumplimiento.

Modes: public_tx | shielded_send | shielded_receive · Compliance: view_key

6. Account Abstraction & UX

  • Patrón estilo ERC-4337: multi-firmas, social recovery, límites/policies.
  • Gas relayers: pagar gas en QSC u otros activos soportados.
  • Wallets móviles: claves PQC y backups con Shamir Secret Sharing.

7. Governance

Gobernanza on-chain minimalista con warm-up y timelock, voto ponderado por stake y quórums razonables.

8. Roadmap

  1. Website + Litepaper
  2. PQC wallet (hybrid keys) + public testnet
  3. VRF + staking; auditoría 1
  4. zk opt-in MVP; auditoría 2
  5. Mainnet (fase 1) + ABIs estables
  6. Rollup-ready L2 bridge + AA avanzada

9. Risks & Disclaimers

  • Técnico: PQC & zk están en evolución.
  • Regulatorio: cumplimiento varía por jurisdicción.
  • Mercado: volatilidad alta; no es consejo financiero.

Documento informativo y sujeto a cambios. El código abierto y las auditorías prevalecen como fuentes de verdad.
